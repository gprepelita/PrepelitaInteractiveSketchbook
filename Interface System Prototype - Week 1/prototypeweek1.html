Weekly Snapshot

Structural Reflection

For this first week's prototype I focused on just coming up with a basic yet clear and easily understandable structure for the main landing page, where all the sketches and other content of the sketchbook is listed. For the deskop version I had two ideas. One is a grid with all the sketches, sort of like a photography portfolio. The title of each page would be centered over a square with some sort of image or background on it that represents the content. The second idea is a card type system where the user would scroll sideways through all the cards of each sketch/page. For the mobile wireframe, I also went with a cards system, but instead of scrolling horizontally, the user would scroll vertically. I think the card system is a bit more modern and visually interesting, but the grid system is more efficient for browsing through a large number of sketches. Some of the issues from my audit that these wireframes address are the lack of structure, hierarchy, navigation and mapping. The organized layout enhances these elements a lot. Some of the heuristics, such as consistency and standards, as well as flexibility and efficieny of use are also emphasized more in this structured layout of the main page. Something that still feels unresolved is the use and placement of imagery, as well as type choice. I need to explore more options for how to visually represent each sketch and make the titles more visually interesting. I also need to decide on a consistent typeface and hierarchy for the text on the page. I might also consider some sort of nav bar or menu at the top.
