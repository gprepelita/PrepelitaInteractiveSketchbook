Interface System Statement

My sketchbook interface system organizes the elements of my sketchbook in a clean and simple way that I think is easy to navigate and understand. Using a mobile-first approach, I designed the interface to be clearly legible at a small size, with large tiles that a user can click through to access each page from the home page. The home page also includes a search bar and filter function to allow users to easily find the content they are looking for. The sketch pages have a consistent layout that is easy to understand, with the title and description of the sketch at the top and the sketch itself taking up the rest of the screen. I have also added a home button on the sketch page so that the user can easily navigate back to the home page. The micro project page includes instructions for how to play the game/sketch, as well as a landscape version of the screen for when the user flips their phone to start the game. I have also included a home button on the micro project page, which was not present in previous iterations. The user can still go back by clicking the back arrow on their browser if they don't want to scroll up from the sketch.

The desktop version of the interface is very similar to the mobile, mainly with the exception of the home page, where I use a different tile system for grouping all the different sketches. Here, they are all arranged in a grid similar to a photo portfolio, that the user can just immediately click on whatever sketch page they want. The filter, search and home functions are all included in the same way as in the mobile version. The micro project page doesn't require a separare landscape version since the game can fit on a desktop screen but I still included a second page to show how the sketch progresses from the main menu to the game screen itself.

Tying in to Norman's principles of interaction, my interface's affordances are mainly all the clickable link/tiles/buttons that allow the user to navigate through the sketchbook. Additionally, there are all the other affordances specific to the micro project, like moving the avatar with the WASD keys and pressing the space bar for hints. The signifiers that I use in my interface are the filter and home buttons, and the search bar. The icons for these are universally recognizable and I think they clearly indicate their function. The mapping of the interface is pretty straightforward, with the home page leading to the sketch pages and micro project page, and the sketch pages having a home button that leads back to the home page. The feedback in my interface is mainly visual, with the user being able to see when they click on a tile or button and are taken to a different page. I also plan on adding more visual feedback on hovering over the tiles when I start including styling into the interface. In the micro project, there is also feedback in the form of the game itself, where the user can see how their actions affect the game state.

My interface system contains several usability heuristics. The first is visibility of system status, which is straightforward on each page and most present in my micro project itself, through informational text. Secondly,a match between the system and the real world is present through the use of universal icons for the filter and home buttons and the shape of the search bar. User control and freedom is afforded throughout. Consistency and standards are again followed through the icons as well as the tile system used on the main page. Recognition rather than recall is also present as the user has clear visibility of and access to the navigational items. Flexibility and efficiency of use is not a focus in my interface but it is somewhat present in my micro project. Finally, minimalist design has been one of the main things I tried to accomplish with the design of this interface, in order to make it very simple to navigate. I have not added error prevention, recognition or diagnosis mechanisms, or any help and documentation. These may come later if necessary. I think this system also includes most of the 8 golden rules of interface design, again with the exception error handling and maybe informative feedback, which I could also improve on in the future. There is consitency in the fonts, layout, and overall design. The system is universally usable as it is designed to be simple and intuitive, with few buttons or menus. The sequence of actions is pretty straightforward, with the user starting on the home page and then clicking through to the sketch pages or micro project page, giving closure. Navigation is easily reversible with the home button. Users are kept in control since there are no unexpected changes in the interface, and users have very little need to remember information.
